第一节 如何跳过安装deb包时的授权验证

首先呢，polkit有一个版本之分，先确认其版本，否则你就会和我一样，花了半夜时间才知道网上讲的大多数都是新版本规则如何添加，but你加了却死活没反应，emmm,,,
命令：pkexec --version

++++++++++++++++++
fpc@fpc-PC:~$ pkexec --version
pkexec version 0.105
++++++++++++++++++

如果你的版本<0.106,那你往下看，反之，你看上面的链接就够了，这篇文章都是基于105版本写的，106之前版本的规则文件是pkla格式，之后的版本是rules
105版本的规则文件以pkla结尾，类似的文件你可以到/var/lib/polkit-1/localauthority/目录中去查看，里面的文件以数字开头，数字大小代表其优先级，有关优先级有下面一段话描述挺好：
polkitd 以字典顺序读取 /etc/polkit-1/rules.d 和 /usr/share/polkit-1/rules.d 目录中的 .rules 文件。如果两个文件采用同样的名称，/etc 中的文件在 /usr 中的文件前面。处理旧的 .pkla 文件时，最先采用的是最后的规则。使用新的 .rules 文件时，首先采用的是第一个匹配的规则。


查看以下目录：

++++++++++++++++++
root@fpc-PC:/etc/polkit-1/localauthority# ls
10-vendor.d 20-org.d 30-site.d 50-local.d 90-mandatory.d
++++++++++++++++++

里面按照优先级分了不同的文件夹，如果你修改的内容没有已存在的相同配置的话，一般是不会冲突的，
新建文件org.kubuntu.qaptworker3.pkla,文件书写要求见附录1：
内容如下（不包括加号部分）：
++++++++++++++++++
[install and uninstall without password]
Identity=unix-group:sudo
Action=org.kubuntu.qaptworker3.commitchanges
ResultAny=no
ResultInactive=no
ResultActive=yes
++++++++++++++++++
将此文件复制到/etc/polkit-1/localauthority/10-vendor.d/文件夹下即可生效，下面来剖析一下这个文件
第一行：[install and uninstall without password] 随便取名，一般体现出这个配置文件的作用即可
第二行：Identity=unix-group:sudo 这里就是说此操作授予sudo权限
第三行：Action=org.kubuntu.qaptworker3.commitchanges 具体的操作
第四行：类似于ResultActive，但适用于任何主题。

第五行：与ResultActive相似，但适用于处于非活动状态的本地会话中的主题。

第六行：在活动的本地会话中与一个或多个给定身份匹配的主题返回的结果。可选指： yes， no， auth_self， auth_self_keep， auth_admin和 auth_admin_keep，这些不同的选项有不同的含义，如果不清楚，请看写在最前。

第二节 如何跳过其他的验证

也就是我们也按照第一节讲的写一个其他操作的pkla规则文件，然后放到指定的目录就可以 再看一下上一节中所说的pkla文件中的action部分，action指的是具体的操作，比如安装或移除软件包的操作id就是org.kubuntu.qaptworker3.commitchanges，所以这一行写作
Action=org.kubuntu.qaptworker3.commitchanges
如果想换成其他的操作，你就要找到这个的action id.怎么找？每一个操作都要对应的policy文件，这个文件在/usr/share/polkit-1/actions目录下，你打开一看，哇，这么多，我怎么知道哪个操作是我要找的，那你执行一下以下命令pkaction
+++++++++++++++++++++++++++++
fpc@fpc-PC:~$ pkaction
com.deepin.api.device.unblock-bluetooth-devices
...
com.deepin.daemon.accounts.disable-auto-login
com.deepin.daemon.accounts.disable-nopass-login
com.deepin.daemon.accounts.enable-auto-login
...
org.kubuntu.qaptworker3.commitchanges
org.kubuntu.qaptworker3.updatecache
org.kubuntu.qaptworker3.writefiletodisk
org.linux.lmt.gui.policy
org.x.xf86-video-intel.backlight-helper
+++++++++++++++++++++++++++++++++++++++
这里会列出来所有需要polkit授权的操作，先从名称查看是不是你要的操作，事实上你无论看不看得懂，都得打开policy去看这个操作的描述信息，还是以安装应用为例，它对应的action id是org.kubuntu.qaptworker3.commitchanges，我们从字面上是很难看出来是做什么的，那我们就去找这个操作的描述，怎么找？使用吐下命令去查找

grep -rwn "org.kubuntu.qaptworker3.commitchanges" /usr/share/polkit-1/actions

找到了org.kubuntu.qaptworker3.policy文件，内容我还是贴一下吧
+++++++++++++++++++++++++++++++++++++++
<?xml version="1.0" encoding="utf-8"?> <!DOCTYPE policyconfig PUBLIC 

"-//freedesktop//DTD PolicyKit Policy Configuration 1.0//EN"
"http://www.freedesktop.org/standards/PolicyKit/1.0/policyconfig.dtd">

<policyconfig> 

 <vendor>Kubuntu</vendor>
 <vendor_url>http://www.kubuntu.org</vendor_url>

 <action id="org.kubuntu.qaptworker3.updatecache">
   <description>Update software sources</description>
   <message>Update software sources</message>
   <defaults>
     <allow_inactive>no</allow_inactive>
     <allow_active>yes</allow_active>
   </defaults>
 </action>
 <action id="org.kubuntu.qaptworker3.commitchanges">
   <description>Install or remove packages</description>
   <description xml:lang="zh_CN">安装或移除软件包需要授权</description>
   <message>Install or remove packages</message>
   <message xml:lang="zh_CN">安装或移除软件包需要授权</message>
   <defaults>
     <allow_inactive>no</allow_inactive>
     <allow_active>auth_admin_keep</allow_active>
   </defaults>
 </action>
 <action id="org.kubuntu.qaptworker3.writefiletodisk">
   <description>Change system settings</description>
   <message>Change system settings</message>
   <defaults>
     <allow_inactive>no</allow_inactive>
     <allow_active>auth_admin_keep</allow_active>
   </defaults>
 </action>
 <action id="org.kubuntu.qaptworker3.cancelforeign">
   <description>Cancel the task of another user</description>
   <message> To cancel someone else's software changes, you need to authenticate.</message>
   <defaults>
     <allow_any>auth_admin</allow_any>
     <allow_inactive>auth_admin</allow_inactive>
     <allow_active>auth_admin</allow_active>
   </defaults>
 </action>

</policyconfig> +++++++++++++++++++++++++++++++++++++++
重点我已经标出来了，不解释，就是这样找的。他的descriptopn就是你安装deb时授权弹窗的标题文本的来源。不信你可以改了试试，再次授权验证时标题也就变了。

附录

1、AUTHORIZATION ENTRY Each group in a .pkla file must have a name that is unique within the file it belongs to. The following keys are are recognized:

Identity

A semi-colon separated list of globs to match identities. Each glob should start with unix-user: or unix-group: to specify whether to match on a UNIX user name or a UNIX group name. Netgroups are supported with the unix-netgroup: prefix, but cannot support glob syntax.

Action

A semi-colon separated list of globs to match action identifiers.

ResultActive

The result to return for subjects in an active local session that matches one or more of the given identities. Allowed values are similar to what can be used in the defaults section of .policy files used to define actions, e.g. yes, no, auth_self, auth_self_keep, auth_admin and auth_admin_keep.

ResultInactive

Like ResultActive but instead applies to subjects in inactive local sessions.

ResultAny

Like ResultActive but instead applies to any subject.

ReturnValue

A semi-colon separated list of key/value pairs (of the form key=value) that are added to the details of authorization result on positive matches.

All keys specified above are required except that only at least one of ResultAny, ResultInactive and ResultActive must be present. The ReturnValue key is optional.
